# Subscription & Billing Logic

## Plans

| Plan | Price | Stripe Price ID |
|---|---|---|
| Free | $0/mo | — |
| Standard | $9.99/mo CAD | `price_1Tduf0DyokC7Tv7bDRAZBk57` |
| Plus | $19.99/mo CAD | `price_1TduyXDyokC7Tv7bKKoeeh1v` |

Billing is recurring monthly until the user cancels. No annual plan currently.

## Subscription States

| `subscriptionStatus` | Behaviour |
|---|---|
| `null` (free plan) | Only Form 8 accessible |
| `active` | Full access |
| `past_due` | Grace period — full access for 7 days |
| `canceled` | Reverts to free plan rules |

Active subscribers are **never** mid-session blocked. Only failed payments (after grace period) or manual cancellations restrict access.

## Stripe Webhook Events Handled

| Event | Action |
|---|---|
| `checkout.session.completed` | Upgrade user to Standard or Plus |
| `invoice.payment_succeeded` | Renew subscription period |
| `invoice.payment_failed` | Set `subscriptionStatus: past_due`, start grace period |
| `customer.subscription.deleted` | Downgrade to free |

## Freemium UX

- "Try it free" — user can fill Form 8 fully
- PDF export shows a lock icon with upgrade prompt
- All other forms show a lock icon with plan name required
- Upgrade flow opens Stripe Checkout in the same tab
